4.1 Что такое Git

В этом видео уроке я расскажу вам, что такое система управления версиями и мы начнем пользоваться системой Git. А в следующих уроках изучим GitHub, будем работать с Git в терминале редактора и графическом интерфейсе.

Git — распределённая система контроля версий. Позволяет отслеживать изменения в файлах, хранить их версии и оперативно возвращаться в любое сохранённое состояние.

Особенности:

  • Каждый разработчик получает полную копию истории проекта на свой локальный компьютер.
  • Все операции выполняются локально без обращения к серверу.
  • Git — основа для популярных платформ совместной разработки, таких как GitHub, GitLab и Bitbucket.

Принцип работы

В отличие от многих других систем контроля версий, которые отслеживают изменения файлов как наборы различий (дельт), Git сохраняет снимки состояния всего проекта в каждый момент времени. Каждый раз при создании коммита система фиксирует состояние файла (делает снимок) и создаёт ссылку на этот снимок. Последующие изменения отражаются через ссылки на более ранние версии файла, нет необходимости снова сохранять файл целиком.

Пример: нужно добавить в проект новую функцию, изменив или доработав существующий код. Для этого создаётся внутри проекта отдельная ветка (branch). Работа в этой ветке не затрагивает основной код — если с новыми изменениями что-то пойдёт не так, основной проект не пострадает.

Структура

Репозиторий Git —хранилище файлов со всей историей изменений проекта. Некоторые компоненты структуры:

  • Рабочая директория — место, где находятся фактические файлы проекта.
  • Индекс (staging area) — область, где подготавливаются изменения для следующего коммита.
  • Коммиты — снимки репозитория в определённый момент времени, включают изменения файлов и метаданные (сообщение коммита, автор и временную метку).


Команды

Некоторые команды для работы с Git:

  • git init — создаёт новый локальный репозиторий.
  • git clone — копирует уже существующий репозиторий со всей историей.
  • git status — отображает актуальное состояние файлов в рабочем каталоге и индексе.
  • git log — отображает список коммитов в порядке от самых новых к более старым, позволяя проследить историю изменений проекта.

Ресурсы для изучения

Для изучения Git можно использовать:

  • Официальную документацию — содержит подробные инструкции по установке Git на различные операционные системы.
  • Интерактивные обучающие платформы — позволяют получить практический опыт работы с Git с помощью упражнений и обратной связи в режиме реального времени.
  • Книги — например, «Git — The Simple Guide», которая предлагает введение в Git, охватывающее основные команды для создания репозиториев, фиксации изменений, ветвления, слияния.
