4.10 Операции с внешними репозиториями в Git

В данном разделе мы начнем изучать операции с внешними репозиториями Git. Мы научимся отправлять данные в репозиторий и получать изменения из репозитория.

Отправка данных во внешний репозиторий в Git

Давайте подготовим и отправим наш первый коммит для нашего тестового внешнего репозитория на сайте GitHub.
Сделаем файл test.txt и проиндексируем его:

ТЕРМИНАЛ

git add test.txt


Выполним коммит:

ТЕРМИНАЛ

git commit -m "Добавлен файл test.txt"


А теперь отправим наш коммит во внешний репозиторий. Для этого нужно использовать команду push, в которой после флага -u через пробел указывается имя удаленного репозитория и имя ветки, которую мы отправляем:

ТЕРМИНАЛ

git push -u Viller main


После выполненения команды, терминал попросит вас ввести ваш логин на GitHub и токен, который мы создали в предыдущих уроках (вводить нужно только первый раз). После ввода токена все ваши коммиты, которых еще нет на сервере, будут отправлены.

Если после этого вы на сайте GitHub зайдете в ваш проект, вы увидите там файлы с вашим кодом.


Сделайте первый коммит вашего проекта и отправьте его во внешний репозиторий. Найдите на GitHub ваш проект и убедитесь, что там появятся изменения.

Внесите изменения в проект. Сделайте еще несколько коммитов. Отправьте ваши изменения во внешний репозиторий.


Короткая команда отправки данных в Git

В предыдущем уроке мы писали следующую команду отправки данных во внешний репозиторий:

ТЕРМИНАЛ

git push -u Viller main


У этой команды есть короткая форма, при использовании которой будет отправлена текущая активная ветка:

ТЕРМИНАЛ

git push



Попробуйте короткую форму отправки.

Отправка в другую ветку в Git

Можно сделать так, чтобы имя ветки на локалке отличалось от имени ветки на сервере. Для этого в команде отправки сначала пишем имя локальной ветки, а потом через двоеточие имя внешней:

ТЕРМИНАЛ

git push origin master:remote_branch


Отправьте данные в другую ветку.

Получение изменений из внешних репозиториев в Git

Итак, в предыдущих уроках мы научились отправлять данные во внешний репозиторий. Представим теперь ситуацию, что данные в нем поменялись. Такая ситуация возникает, когда над проектом работают несколько человек и каждый из них отправляет данные в этот общий репозиторий. Либо может быть ситуация, когда вы работаете на разных компьютерах, например, дома и на работе. На работе вы отправляете изменения, а дома хотите их получить на свой домашний компьютер.

Давайте сымитируем такую ситуацию. Для этого, на сайте GitHub зайдите в файлы проекта и отредатируйте какой-нибудь файл, внеся в него изменения. Давайте получим эти изменения на наш локальный компьютер.

Это делается с помощью команды fetch, после которой указывается имя внешнего репозитория:


ТЕРМИНАЛ

git fetch Viller


Данная команда скопирует все изменения, сделанные во внешнем репозитории с момента вашего последнего обращения к нему. При этом изменения не появятся сразу в наших файлах, а попадут в специальную ветку. Имя этой ветки будет строится по следующему принципу: имя внешнего репозитория, а потом через слеш имя ветки. В нашем случае это будет Viller/main.

Чтобы изменения появились в файлах нашего проекта, нужно влить изменения в нашу ветку. Для этого, находясь в ветке main, выполним команду на слияние:

ТЕРМИНАЛ

git merge Viller/main


На сайте GitHub вручную внесите изменения в ваш проект. Получите эти изменения в свой локальный репозиторий

Стягивание данных из внешних репозиториев в Git

Немного неудобно то, что данные из внешнего репозитория не сливаются сразу с нужной веткой и для этого нужно делать еще некоторые телодвижения. К счастью в Git можно получить изменения из внешнего репозитория, сразу же выполнив слияние. Это делается с помощью команды pull (стянуть), после которой указывается имя внешнего репозитория и имя ветки:

ТЕРМИНАЛ

git pull Viller main


Получите изменения в свой локальный репозиторий с одновременным слиянием.

Короткая команда pull в Git

В предыдущем уроке мы пользовались командой pull следующем образом:

ТЕРМИНАЛ

git pull Viller main



У этой команды есть короткая форма, при использовании которой изменения вольются в соответствующую ветку:

ТЕРМИНАЛ

git pull


Воспользуйтесь коротким вариантом команды.

Информация о коротких командах в Git

Как вы уже знаете, команды push и pull имеют короткие формы. При необходимости можно посмотерть локальные ветки и соответствующие им внешние. Это делается с помощью известной вам команды:

ТЕРМИНАЛ

git remote show Viller



В результате мы увидим следующую информацию:

ТЕРМИНАЛ

Local branch configured for 'git pull':
master merges with remote master
Local ref configured for 'git push':
master pushes to master




Удаление внешних веток в Git

После завершения работы над проектом нам может понадобиться удалить ненужные больше внешние ветки. В этом случае мы можем использовать команду push с флагом --delete, после которого нужно указать имя внешнего репозитория и имя ветки:

ТЕРМИНАЛ

git push --delete Viller main


Клонирование внешнего репозитория Git

Проекты, размещенные на GitHub, можно клонировать. Это значит, что вы можете стянуть себе весь код этого проекта. Это может быть как ваш проект, так и чужой. После клонирования вы сможете вносить изменения в этот проект у себя на локалке. Для клонирования необходимо иметь ссылку на проект. Пусть эта ссылка у вас есть. В этом случае клонировать проект себе можно с помощью команды clone, после которой указывается ссылка на проект:

ТЕРМИНАЛ

git clone https://github.com/Viller/ITDoctor.git


В начале процесса клонирования создается пустой каталог с именем вашего проекта, затем в нем инициализируется репозиторий .git. В него копируются все файлы из исходного проекта.

Если мы хотим, чтобы имя нашего клона отличалось от названия исходной папки, то в конце команды добавляем новое имя:

ТЕРМИНАЛ

git clone https://github.com/Viller/ITDoctor.git my-new_project


Клонинируйте свой тестовый проект, размещенный на GitHub, в новую папку.
